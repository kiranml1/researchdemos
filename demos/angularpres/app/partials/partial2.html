• AngularJS applications cannot be nested within each other
  • AngularJS Controller should not contain DOM Manipulation, Would be always better to chose it for only business Logic and depend it with other concepts
  • Bootstrapping Application would be better with angular.module rather than ngApp Directive
  • A common mistake is to declare the controller again using ng-controller in the template itself. This will cause the controller to be attached and executed twice.
  • Using ngBind is Preferrable Option in contrast to ngCloak for hiding uncomplied content of Angular Expressions
  • Using Angular markup like {{hash}} in an href attribute will make the link go to the wrong URL if the user clicks it before Angular has a chance to replace the {{hash}} markup with its value
  • ngPluralize is the core options to use for localization & internalization
  • Angular JS has a CSP Included subject - Content Security Policy, which allows to restrict some security threats with in the application from client side
  • Use ngModel directive whereever possible to maintain the state control of the application as such with communication as a statemachine
  • Angular JS Apps if they are multiplied with such no in single app it is better to use manual bootstrapping to avoid conficts.
  • Angular JS is a MV* or MVW Framework - Model View Whatever Framework
  • Be Focused to write ng-app/ngApp are same, coming to custom directives - it is same as custom-dir/customDir
